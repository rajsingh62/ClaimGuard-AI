# AI Task Prompt: Complete Production Setup for ClaimGuard AI

**Role & Objective:**
You are an expert Full-Stack AI Developer. Your objective is to seamlessly merge our gorgeous UI templates (located in the `stitch` folder) with our powerful Llama3 OCR-powered car insurance evaluation backend (currently in `ClaimGuard_AI_Kaggle_v10.ipynb` or `build_notebook_v10.py`). 

The final result must be a polished, production-ready, lightning-fast integrated application running locally. No mock data, no "demo" buttons. Everything must be functional.

**Step-by-Step Requirements:**

### 1. Unified FastApi Application
- Create a `main.py` using **FastAPI** that serves as the core backend server.
- The server should mount the `stitch` folders (`hero_experience`, `analysis_lab`, `results_dashboard`) as static frontend files.
- Setup the following routes mapping to the HTML files:
  - GET `/` -> Serves `stitch/hero_experience/code.html` (The landing page).
  - GET `/lab` -> Serves `stitch/analysis_lab/code.html` (The document upload & processing page).
  - GET `/results` -> Serves `stitch/results_dashboard/code.html` (The financial breakdown page).

### 2. Backend Evaluation Endpoint (`/api/evaluate`)
- Extract the core logic from `build_notebook_v10.py` into `main.py` (or a dedicated `engine.py` imported into `main.py`):
- Implement the `extract_text_from_file` mechanism using the existing `OCR.space` API integration and PyPDF2/pdf2image libraries.
- Implement the `car_claim_evaluator` using the `call_ollama` function pointing to `http://localhost:11434/api/generate` (Llama 3 8B model).
- Create a FastAPI endpoint `POST /api/evaluate` that accepts two `UploadFile` parameters (`policy_file` and `bill_file`).
- This endpoint returns the exact structured JSON generated by Llama3.

### 3. Wiring up the "Real" Frontend Experience
Make the following dynamic enhancements to the HTML files using Vanilla Javascript:

**Home Page (`hero_experience/code.html`):**
- Make the "Analyze My Policy" button functional so it navigates the user directly to the `/lab` page.

**Analysis Lab (`analysis_lab/code.html`):**
- Implement real Drag-and-Drop functionality for the two required files: (1) The Insurance Policy and (2) The Repair Bill.
- Add an upload button or intercept form submission.
- Once submitted, show the existing beautiful processing animations (rotating data rings, percentage updates).
- Make a background `fetch` to `/api/evaluate` with the FormData.
- When the API successfully returns, store the resulting JSON in `localStorage` under `claimguard_results` and instantly redirect the user to `/results`.

**Results Dashboard (`results_dashboard/code.html`):**
- Parse the JSON results from `localStorage`.
- Dynamically update the main financial metrics based on the JSON payload:
  - Replace `$1,250,000` with the `total_bill_amount` or `total_covered_amount`.
  - Update the dynamic Risk Gauge according to the percentage of unpayable/uncovered items.
- Replace the static "Detected Policy Anomalies" (The Clauses section) with real items dynamically generated from the JSON's `breakdown` array and `summary` reasons. Render them inside high-tech cards matching the existing design system.

**Quality Constraints:**
- **Polished & Fast:** Interactions should be silky smooth. Use background asynchronous processing so the UI does not freeze during OCR.
- **Production-Ready:** Ensure all buttons point to real internal links or valid actions. Remove placeholder generic "#" hrefs.
- **Flawless Design:** Do not break the Tailwind CSS styling. Only append or modify text content and innerHTML of dynamic containers.

Execute these steps comprehensively to convert the static conceptual design into a fully-functional, real-world AI tool.
